# Computer Science A-Level Battleships Project

## Overview:
As part of my Computer Science A-Level, I had to work on an extended programming project. This involved me having to conduct research about already existing products and then formulate my own designs. 

I decided to create a version of Battleships, which could be played on desktop PCs. By using a variety of different data structures, file handling and an Insertion Sort algorithm - I was able to create a version of Battleships. 

The final version of the program allows:
- Players are able to play against the computer (CPU) or against another player (via local co-op). 
- Players can compete to try to earn the highest score to earn their place on the leaderboard.
- Players can save their games and load them to continue playing another time.

----

## Technical Details
- Data Structures: By using different data structures (including 2D arrays, lists and records) I managed to manage the players' ships, grids and turns.
- File Handling: By using file handling, I implemented a saving and loading system for each gamemode and storing the leaderboard data.
- Algorithms: By using an Insertion Sort, the player's scores are sorted and the top scores are displayed on the leaderboard.
- Game Logic: The game logic consists of: turn-based gameplay, firing hit and miss checking and tracking, generating CPU ships in a random location and allowing players to place their own ships.
- Object-orientated Programming (OOP): By using the OOP programming paradigm, I was able to write more robust code by using encapsulation which could be used more efficiently by reusing components.
- Software: The game was scripted in C# by using Visual Studio.

----

## Features
- Player vs CPU Gamemode: Single-player vs CPU with AI logic.
- Player vs Player Gamemode: Local multiplayer.
- Scoring system with a functional leaderboard that displays the top ten highest scores.
- Save and load functionality for long-term play.
- A clear and simple user-interface (UI)
